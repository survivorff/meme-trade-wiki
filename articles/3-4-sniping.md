# 狙击（Sniping）：抢跑的艺术与代价

> 在 Token 创建的第一个区块就买入，听起来很美好。但这场速度竞赛的真相远比你想象的残酷。

## 表面认知

"用狙击机器人第一个买入，然后等涨了卖出。" 理论上没错，但实际操作中有无数的坑。

## 真实情况

### 狙击是怎么工作的

```
Pump.fun 上有人创建了新 Token
    │
    ├── 链上事件被广播（~400ms）
    │
    ├── 狙击机器人检测到事件（~10-50ms 处理时间）
    │
    ├── 机器人构建买入交易
    │     ├── 计算买入金额
    │     ├── 设置滑点
    │     └── 打包成 Jito Bundle
    │
    ├── 通过 Jito 提交 Bundle（~100-200ms）
    │
    └── 等待下一个区块确认（~400ms）
```

从 Token 创建到狙击交易上链，整个过程在 1-2 秒内完成。

### 狙击的层级

不是所有狙击都一样。按速度和技术水平分：

**第一层：专业机器人（<1 秒）**
- 自建节点，直接连接验证者
- 使用 Staked Connection 获得优先传输
- Jito Bundle 保证执行顺序
- 投入：数万美元的基础设施成本

**第二层：平台内置狙击（1-5 秒）**
- 使用 Axiom、GMGN 等平台的狙击功能
- 平台代为构建和提交交易
- 速度取决于平台的基础设施质量

**第三层：手动快速交易（5-30 秒）**
- 看到新币后手动点击买入
- 在 Meme 币的时间尺度上，这已经算"慢"了

### 狙击的风险

**1. 大多数新币归零**
- 你狙击的 Token 有 98%+ 的概率永远不会毕业
- 即使你是第一个买入的，如果没有后续买盘，你的投入就归零了

**2. 狙击竞争**
- 你不是唯一的狙击手
- 多个机器人同时狙击同一个 Token，会在第一时间推高价格
- 你以为你买在了最低点，实际上可能已经被其他机器人推高了

**3. Dev 钱包风险**
- Token 创建者可能自己就是最大的持有者
- 他们可以在价格被狙击手推高后立即卖出（Rug Pull）
- 狙击手反而成了 Dev 的退出流动性

**4. 蜜罐（Honeypot）**
- 有些 Token 被设计成只能买不能卖
- 狙击机器人如果没有做安全检查，可能买入后无法卖出

### 狙击的经济学

假设你每次狙击投入 0.1 SOL：

```
每次狙击成本：
  - 本金：0.1 SOL
  - Jito Tip：~0.005 SOL
  - Priority Fee：~0.001 SOL
  - 平台手续费：~0.001 SOL
  总成本：~0.107 SOL

假设：
  - 2% 的 Token 能毕业并有利润
  - 毕业的 Token 平均能赚 5 倍
  
100 次狙击：
  - 总投入：10.7 SOL
  - 2 次成功 × 0.5 SOL 利润 = 1.0 SOL
  - 98 次失败 × 0.107 SOL = 10.486 SOL 损失
  - 净亏损：~9.5 SOL
```

**除非你的选币能力远超平均水平，否则盲目狙击是亏钱的。**

### 平台方怎么看狙击

作为平台方，我们对狙击有复杂的态度：
- 狙击功能是用户需求，不提供就会流失用户
- 但狙击也带来了很多问题（用户亏损后的投诉、安全风险）
- 平台需要在"提供工具"和"保护用户"之间找平衡

## 经验教训

1. **狙击不是稳赚策略** —— 大多数狙击交易是亏损的
2. **速度不是唯一因素** —— 选币能力比速度更重要
3. **做好安全检查** —— 至少检查合约权限、持仓分布、创建者历史
4. **控制仓位** —— 每次狙击只用很小的金额，做好全部亏完的准备
