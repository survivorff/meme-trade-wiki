# 怎么发现新币：Token Radar 的工作原理

> 在 Meme 币市场，发现得早就是赚钱的一半。但"早"到底意味着什么？

## 表面认知

"刷 DEXScreener 看新币就行了。" 当你在 DEXScreener 上看到一个 Token 时，专业玩家可能已经买入几分钟甚至几秒了。

## 真实情况

### 信息传播链

一个新 Token 从创建到被你看到，经历了这样的传播链：

```
Token 创建（第 0 秒）
    │
    ├── 链上事件触发（第 0.1 秒）
    │     └── 狙击机器人自动买入
    │
    ├── 交易平台检测到（第 1-5 秒）
    │     └── 平台用户看到新币列表
    │
    ├── DEXScreener/Birdeye 收录（第 10-60 秒）
    │     └── 数据平台用户看到
    │
    ├── Telegram 群/Twitter 传播（第 1-10 分钟）
    │     └── 社交媒体用户看到
    │
    └── KOL 喊单（第 10-60 分钟）
          └── 大众散户看到
```

**你在信息链上的位置，直接决定了你的买入价格。**

### 各种发现方式的时效性

| 发现方式 | 延迟 | 适合谁 |
|---------|------|--------|
| 链上事件监听（自建） | <1 秒 | 专业开发者/机器人 |
| 交易平台新币列表 | 1-10 秒 | 平台用户 |
| DEXScreener 新币页 | 10-60 秒 | 数据平台用户 |
| Telegram 群消息 | 1-10 分钟 | 社群活跃用户 |
| Twitter/X 推文 | 10-60 分钟 | 社交媒体用户 |
| KOL 推荐 | 30 分钟-数小时 | 大众散户 |

### 交易平台的 Token 发现机制

作为平台方，我们是这样发现新 Token 的：

**1. 链上事件监听**
- 监听 Pump.fun 合约的 Token 创建事件
- 监听 Raydium/PumpSwap 的流动性池创建事件
- 实时解析每个新区块中的相关交易

**2. 过滤和评分**
- 不是所有新 Token 都值得展示
- 平台会根据多个维度进行过滤和评分：
  - 创建者历史（是否有成功的 Token？是否有 Rug Pull 记录？）
  - 初始买入情况（有多少独立地址在买？）
  - 社交媒体热度（Twitter 提及量、Telegram 讨论量）
  - 合约安全性（是否有可疑的权限设置？）

**3. 排序和展示**
- 按热度、交易量、价格变化等维度排序
- 突出显示"趋势"Token
- 提供过滤器让用户自定义筛选条件

### Smart Money 追踪

另一种发现方式是追踪"聪明钱包"：

- 平台标记历史上盈利率高的钱包地址
- 当这些地址买入新 Token 时，发出提醒
- 用户可以选择自动跟单

但这里有一个重要的陷阱 —— 详见下一篇。

## 经验教训

1. **速度是分层的** —— 不同的发现方式有不同的延迟，选择适合你的
2. **不要追求最快** —— 除非你是专业的机器人运营者，否则和机器人比速度没有意义
3. **过滤比发现更重要** —— 每天数万个新 Token，关键是筛选出值得关注的
4. **信息源的质量决定了你的胜率** —— 建立自己的信息管道比依赖单一平台更可靠
