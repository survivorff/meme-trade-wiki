# 平台安全：你没想到的攻击面

> 交易平台是黑客的高价值目标。用户资金、API 接口、运维系统，每一个都是潜在的攻击面。

## 表面认知

"把私钥保管好就行了。" 私钥安全只是冰山一角。

## 真实情况

### 攻击面全景

```
┌─────────────────────────────────────────┐
│              外部攻击面                    │
│  API 攻击 · DDoS · 钓鱼 · 社会工程       │
├─────────────────────────────────────────┤
│              应用层攻击面                  │
│  业务逻辑漏洞 · 权限绕过 · 注入攻击       │
├─────────────────────────────────────────┤
│              基础设施攻击面                │
│  服务器入侵 · 供应链攻击 · 内部人员        │
├─────────────────────────────────────────┤
│              链上攻击面                    │
│  私钥泄露 · 交易重放 · 合约漏洞           │
└─────────────────────────────────────────┘
```

### 关键安全威胁

**1. 私钥安全（最高优先级）**
- 托管钱包的私钥是最高价值的攻击目标
- 泄露 = 用户资金全部丢失
- 防护：HSM/KMS 加密存储、签名服务隔离、多层权限控制

**2. API 安全**
- 交易 API 可能被滥用（刷单、套利）
- 需要：限流、身份验证、请求签名、异常检测

**3. 前端安全**
- 钓鱼网站模仿平台界面
- XSS 攻击窃取用户 Session
- 防护：CSP 策略、域名监控、用户教育

**4. 内部人员风险**
- 有权限访问私钥或敏感数据的员工
- 防护：最小权限原则、操作审计、多人审批

**5. 供应链攻击**
- 依赖的第三方库被植入恶意代码
- npm/cargo 包被劫持
- 防护：依赖审计、锁定版本、安全扫描

**6. DDoS 攻击**
- 在关键时刻（热门 Token 发射）发起 DDoS
- 平台无法服务用户，竞争对手获利
- 防护：CDN、WAF、弹性扩容

### 安全最佳实践

| 领域 | 措施 |
|------|------|
| 密钥管理 | HSM/KMS、签名服务隔离、多签 |
| 访问控制 | 最小权限、MFA、定期审计 |
| 网络安全 | VPN、防火墙、网络隔离 |
| 代码安全 | 代码审计、依赖扫描、安全测试 |
| 运维安全 | 日志审计、告警、应急预案 |
| 用户安全 | 2FA、提现白名单、异常登录检测 |

### 安全事件响应

当安全事件发生时：
1. **检测**：自动化监控发现异常
2. **遏制**：立即冻结受影响的功能/账户
3. **分析**：确定攻击范围和影响
4. **修复**：修补漏洞，恢复服务
5. **通知**：透明告知用户
6. **复盘**：分析根因，防止再次发生

## 经验教训

1. **安全是持续的过程，不是一次性的工作**
2. **假设你会被攻击** —— 问题不是"会不会"而是"什么时候"
3. **最小化攻击面** —— 能不暴露的就不暴露
4. **安全审计不能省** —— 定期请第三方审计
