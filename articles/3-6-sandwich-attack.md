# 你的交易为什么被夹了：三明治攻击完全解读

> 你以为你在和市场交易，实际上有人在你的交易前后各插了一刀。

## 表面认知

"我的交易怎么成交价比预期差这么多？" 你可能被三明治攻击了。

## 真实情况

### 三明治攻击原理

```
你提交了一笔买入交易（买 Token X，愿意接受 5% 滑点）
    │
    ↓
攻击者看到了你的交易（通过监控交易流）
    │
    ├── 步骤 1：攻击者先买入 Token X（推高价格）  ← Front-run
    ├── 步骤 2：你的交易执行（以更高的价格买入）  ← 你的交易
    └── 步骤 3：攻击者卖出 Token X（获利）        ← Back-run
```

你被"夹"在了攻击者的两笔交易中间，就像三明治的馅料。

### 在 Solana 上怎么实现的

Solana 没有传统意义上的公开 Mempool，但三明治攻击仍然存在：

**攻击路径：**
1. 攻击者运营自己的验证者节点，或与验证者合作
2. 当用户的交易到达验证者时，攻击者可以看到交易内容
3. 攻击者构建 Bundle：[自己的买入] + [用户的交易] + [自己的卖出]
4. 通过 Jito 提交 Bundle，保证三笔交易的执行顺序

**关键数据：**
- 2024-2025 年，三明治攻击从 Solana 用户身上提取了约 $5 亿
- 高峰期，93% 的三明治攻击使用跨 Slot 技术
- 2025 年后，Solana 基金会和 Jito 采取措施，三明治攻击下降了 93%

### 你怎么知道自己被夹了

检查方法：
1. 查看你的交易在区块中的位置
2. 如果你的交易前后紧邻着同一个地址的买入和卖出，你就被夹了
3. 工具：Solscan、SolanaFM 可以查看交易详情

典型特征：
- 实际成交价格明显差于预期
- 同一个区块中，你的交易前后有可疑的大额交易
- 滑点消耗接近你设置的最大值

### 怎么防护

**1. 降低滑点设置**
- 滑点设置越高，攻击者的利润空间越大
- 但设太低交易可能失败
- 建议：根据 Token 的流动性动态调整

**2. 使用 Jito Bundle**
- 通过 Jito 提交你的交易，可以避免被插入其他交易
- Jito 的 DontFront 功能专门用于防三明治

**3. 拆分大额交易**
- 大额交易更容易被盯上（利润空间更大）
- 拆成多笔小额交易可以降低被攻击的概率

**4. 使用平台的 MEV 保护功能**
- 部分交易平台内置了 MEV 保护
- 自动通过 Jito Bundle 提交交易
- 自动优化滑点设置

### Solana 生态的反三明治措施

- **Jito DontFront**：保证 Bundle 内的交易不会被插入其他交易
- **Jito BAM**：新的交易排序机制，减少有害 MEV
- **Solana 基金会**：对参与三明治攻击的验证者进行处罚
- **效果**：2025 年三明治攻击盈利能力下降了 60-70%

## 经验教训

1. **三明治攻击是结构性问题** —— 只要有交易排序的控制权，就有 MEV 提取的可能
2. **防护措施在不断进化** —— Jito DontFront 和 BAM 显著改善了情况
3. **大额交易要格外小心** —— 金额越大，被攻击的动机越强
4. **理解你的交易是怎么被执行的** —— 知道风险才能管理风险
