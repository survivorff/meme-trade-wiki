# 如何识别 Rug Pull：从合约到链上行为的红旗信号

> 在 Meme 币市场，Rug Pull 不是意外，而是常态。学会识别红旗信号是生存的基本技能。

## 表面认知

"Rug Pull 就是项目方跑路。" 没错，但跑路的方式有很多种，有些非常隐蔽。

## 真实情况

### Rug Pull 的类型

**1. 硬 Rug（Hard Rug）—— 直接撤走流动性**
- 创建者移除 DEX 上的流动性池
- Token 瞬间无法交易，价格归零
- 最粗暴但也最容易识别（LP 是否锁定/销毁）

**2. 软 Rug（Soft Rug）—— 缓慢出货**
- 创建者或内部人持有大量 Token
- 在价格上涨过程中分批卖出
- 不会瞬间归零，但价格持续下跌
- 更难识别，因为看起来像"正常的获利了结"

**3. 蜜罐（Honeypot）—— 只能买不能卖**
- 合约中设置了限制卖出的逻辑
- 用户可以买入但无法卖出
- 在 Pump.fun 的标准合约中较少见，但在自定义合约中常见

**4. Mint Rug —— 无限增发**
- 创建者保留了 Mint 权限
- 可以随时增发新 Token 稀释所有持有者
- 然后卖出增发的 Token 获利

### 红旗信号清单

**合约层面：**

| 信号 | 风险等级 | 说明 |
|------|---------|------|
| Mint 权限未撤销 | 🔴 高 | 创建者可以无限增发 |
| Freeze 权限未撤销 | 🔴 高 | 创建者可以冻结任何人的 Token |
| LP Token 未销毁/未锁定 | 🔴 高 | 创建者可以随时撤走流动性 |
| 合约有 blacklist 功能 | 🟡 中 | 可以阻止特定地址卖出 |
| 合约可升级 | 🟡 中 | 逻辑可以被修改 |

**持仓分布层面：**

| 信号 | 风险等级 | 说明 |
|------|---------|------|
| 单一地址持有 >10% | 🔴 高 | 大户可以砸盘 |
| 前 10 地址持有 >50% | 🔴 高 | 筹码过于集中 |
| 多个关联地址分散持仓 | 🟡 中 | 可能是同一人用多个钱包 |
| 创建者钱包持仓比例高 | 🔴 高 | Dev 可能随时卖出 |

**行为层面：**

| 信号 | 风险等级 | 说明 |
|------|---------|------|
| 创建者有 Rug Pull 历史 | 🔴 高 | 查看创建者钱包的历史 |
| 大量小额买入制造假活跃 | 🟡 中 | 可能是刷量 |
| 社交媒体账号是新建的 | 🟡 中 | 缺乏可信度 |
| 承诺不切实际的回报 | 🔴 高 | 经典骗局信号 |

### 平台的安全检测

交易平台通常会提供自动化的安全评分：
- 自动检查合约权限（Mint、Freeze、LP 状态）
- 分析持仓分布
- 检查创建者历史
- 给出风险评级

但要注意：**平台的安全检测不是万能的。** 新的 Rug Pull 手法不断出现，自动化检测总是滞后的。

### Pump.fun 的反 Rug 机制

Pump.fun 的标准流程有一些内置的保护：
- Bonding Curve 阶段没有传统的流动性池，不存在"撤走流动性"
- 毕业时 LP Token 被自动销毁
- 创建者不能在 Bonding Curve 阶段获得特殊的 Token 分配

但这不意味着完全安全：
- 创建者可以在 Bonding Curve 阶段用多个钱包大量买入
- 毕业后在 DEX 上卖出获利
- 这种"软 Rug"很难被自动检测到

## 经验教训

1. **没有 100% 安全的 Meme 币** —— 即使通过了所有安全检查
2. **多维度交叉验证** —— 不要只看一个指标，综合判断
3. **如果看起来太好了，那可能就是骗局** —— 保持怀疑
4. **只投入你能承受全部损失的金额** —— 这是最根本的风险管理
