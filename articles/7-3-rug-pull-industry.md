# Rug Pull 产业链：从发币到跑路的完整流程

> Rug Pull 不是个别事件，而是一条成熟的产业链。

## 表面认知

"Rug Pull 就是有人发了个币然后跑路了。" 实际上，很多 Rug Pull 是有组织、有流程、可复制的"生意"。

## 真实情况

### 产业链全景

```
工具层：发币脚本、刷量机器人、社交媒体账号
    │
    ▼
执行层：发币 → 造势 → 拉盘 → 出货 → 转移资金
    │
    ▼
变现层：混币器 → CEX 出金 → 法币
```

### 一次典型 Rug Pull 的完整流程

**第一步：准备（1-2 天）**
- 创建多个新钱包（用于分散持仓）
- 准备 Token 的名称、图片、故事（通常蹭热点）
- 准备社交媒体账号（可能是购买的老号）
- 准备刷量脚本

**第二步：发币（几分钟）**
- 在 Pump.fun 上创建 Token
- 用准备好的多个钱包在第一时间买入
- 控制 20-40% 的流通量

**第三步：造势（几小时到几天）**
- 在 Twitter/Telegram 上发布 Token 信息
- 付费让 KOL 转发或喊单
- 用机器人刷交易量和社交媒体互动
- 在各种 Meme 币群里推广

**第四步：拉盘（几小时）**
- 协调买入推高价格
- 散户看到价格上涨开始 FOMO 买入
- 价格可能涨 10-100 倍

**第五步：出货（几分钟到几小时）**
- 分批从多个钱包卖出
- 利用散户的买盘作为退出流动性
- 可能在短时间内砸盘，也可能缓慢出货

**第六步：资金转移**
- 将获利的 SOL 转移到新钱包
- 通过混币器或跨链桥混淆资金来源
- 最终通过 CEX 兑换为法币

### 规模化运作

一些 Rug Pull 团队会规模化运作：
- 每天发射多个 Token
- 大多数失败（没有吸引到足够的散户）
- 少数成功的利润覆盖所有成本
- 本质上是一种"概率游戏"

### 进化的手法

随着用户安全意识提高，Rug Pull 的手法也在进化：

**传统手法（容易识别）：**
- 直接撤走流动性
- 单一大户砸盘

**进化手法（更难识别）：**
- 使用数十个关联钱包分散持仓
- 缓慢出货而不是一次性砸盘
- 先建立一定的社区信任再出货
- 利用合约的隐藏功能（如可升级合约）

### 受害者画像

最容易被 Rug Pull 的人群：
1. 新手：不知道怎么检查合约安全性
2. FOMO 驱动者：看到价格涨就追
3. KOL 粉丝：盲目相信 KOL 推荐
4. 贪婪者：追求百倍千倍回报，忽视风险

### 防护建议

1. **永远做安全检查** —— 使用 RugCheck 等工具
2. **不要 FOMO** —— 已经涨了很多的 Token 风险更高
3. **小仓位** —— 每次投入不超过你能承受损失的金额
4. **独立判断** —— 不要因为别人推荐就买入

## 经验教训

1. **Rug Pull 是这个市场的结构性问题** —— 低门槛发币 + 匿名性 = Rug Pull 的温床
2. **技术手段可以降低风险但无法消除** —— 安全检测总是滞后于新的攻击手法
3. **教育是最好的防护** —— 理解风险才能管理风险
4. **如果回报看起来太好了，那可能就是陷阱**
